  • 2.2.1 ความหมายของชื่อผลงาน:

  • 2.2.3 แนวคิดหลักในการส่งเสริมและสนับสนุนการบริหารจัดการสถานศึกษา:
    ใช้นวัตกรรม AI (Predictive Modeling) เพื่อทำนายกลุ่มเสี่ยงที่อาจหลุดออกจากระบบการศึกษาล่วงหน้า บูรณาการโดยการประมวลผลข้อมูลจากสถิติการมาเรียนและเกรดเฉลี่ยของผู้เรียน

ส่วนที่ 3 การวิเคราะห์สภาพปัญหาและความจำเป็น (21 คะแนน)

3.1 การวิเคราะห์สภาพระบบบริหารจัดการสถานศึกษา (6 คะแนน)

ปัจจุบันข้อมูลของนักเรียนกระจายตัวอยู่ในหลายแพลตฟอร์ม เช่น ระบบ SGS, ระบบ DMC รวมถึง Checklist ที่เป็นกระดาษ ปัญหาที่เกิดขึ้นจากการใช้งานจริงคือกระบวนการเหล่านี้ทำให้ครูที่ปรึกษาต้องสูญเสียเวลาไปกว่า 8 ชั่วโมงต่อสัปดาห์ในการทำเอกสาร ข้อจำกัดที่สำคัญอีกประการคือผู้บริหารไม่สามารถรับทราบข้อมูลได้อย่างทันท่วงที ทำให้มองเห็นภาพรวมปัญหาของผู้เรียนล่าช้าไปเกิน 1 เดือน

3.2 ความชัดเจนของปัญหาและเหตุผลในการเลือกใช้นวัตกรรมปัญญาประดิษฐ์ (AI) (6 คะแนน)

  • 3.2.1 ปัญหาการจัดการข้อมูลที่กระจายตัวในหลายแพลตฟอร์มและเอกสารกระดาษ ส่งผลให้เกิดความซ้ำซ้อนและครูต้องใช้เวลากว่า 8 ชั่วโมงต่อสัปดาห์ในการจัดการเอกสาร
  • 3.2.2 โครงการได้นำเทคนิค Predictive Modeling มาทำนายกลุ่มเสี่ยงล่วงหน้า ทำงานควบคู่กับการประมวลผลข้อมูลสถิติการมาเรียนและเกรดเฉลี่ย มีการใช้งาน Automated Insights เพื่อช่วยสรุปผลสารสนเทศอัตโนมัติ
  • 3.2.3 นวัตกรรม AI สามารถสร้างข้อมูล KPI ที่สำคัญให้แก่ผู้บริหารได้แบบ Real-time ข้อมูลเชิงลึกนี้สนับสนุนการตัดสินใจเชิงนโยบาย โดยสามารถลดความผิดพลาดในการตัดสินใจได้ร้อยละ 90

3.3 เหตุผลและความจำเป็นในการเลือกใช้นวัตกรรมปัญญาประดิษฐ์ (AI) (9 คะแนน)

  • เพื่อลดระยะเวลาการเข้าถึงข้อมูลของผู้บริหารและครู โดยลดเวลาการสืบค้นข้อมูลจากเดิม 1 ชั่วโมง เหลือเพียง 30 วินาที
  • เพื่อลดภาระงานเอกสารที่ซ้ำซ้อนของครูที่ปรึกษา ซึ่งแต่เดิมต้องใช้เวลาทำเอกสารถึง 8 ชั่วโมงต่อสัปดาห์
  • เพื่อเพิ่มประสิทธิภาพการตัดสินใจเชิงนโยบายของผู้บริหาร โดยมีเป้าหมายในการลดความผิดพลาดในการตัดสินใจได้ร้อยละ 90 ด้วยการดู KPI แบบ Real-time
  • เพื่อแก้ไขปัญหาความล่าช้าในการติดตามผู้เรียน โดยใช้ AI ทำนายกลุ่มเสี่ยงล่วงหน้าแทนระบบเดิมที่ผู้บริหารเห็นปัญหาล่าช้าเกิน 1 เดือน

ส่วนที่ 4 การออกแบบนวัตกรรมและกระบวนการดำเนินงาน (30 คะแนน)

4.1 การพัฒนาระบบบริหารจัดการสถานศึกษา (10 คะแนน)

  • 4.1.1 โครงสร้างระบบข้อมูล: มีการพัฒนาระบบเชื่อมต่อข้อมูล (Data Pipeline) ดึง Data Source มาจากระบบ SGS และ DMC API
  • 4.1.2 โมดูลนวัตกรรมปัญญาประดิษฐ์ (AI) ที่พัฒนา: ใช้ AI Engine ที่พัฒนาด้วยภาษา Python เพื่อทำการวิเคราะห์เรียนรู้ของเครื่อง (ML Analysis) ใช้โมเดล Predictive Modeling ระบบประมวลผลแบบ Automated Insights ผ่าน Google Apps Script
  • 4.1.3 ระบบ Dashboard และการรายงาน: แสดงผล Visualization ของข้อมูลผ่าน Looker Studio Dashboard
  • 4.1.4 ขั้นตอนการดำเนินงาน: แบ่งไทม์ไลน์เป็น 3 ระยะ
        - เดือนที่ 1-2: พัฒนาระบบเชื่อมต่อข้อมูล (Data Pipeline)
        - เดือนที่ 3-4: สร้างโมเดล AI และออกแบบ Dashboard สำหรับผู้บริหาร
        - เดือนที่ 5-6: ทดสอบระบบ (UAT) และขยายผลการใช้งานจริง

4.2 ความเป็นไปได้ในการพัฒนา ทดลองใช้ และประเมินผล (10 คะแนน)

  • 4.2.1 ความพร้อมด้านทรัพยากรและเทคโนโลยี: เครื่องมือที่ใช้ประกอบด้วย SGS/DMC API, Python & ML Analysis, Google Apps Script และ Looker Studio ด้านบุคลากร ทีมพัฒนามีทักษะและผ่านการอบรม Python for AI ด้านทรัพยากร โรงเรียนสนับสนุน Server และระบบ Google Workspace แบบ Enterprise ให้ใช้งาน
  • 4.2.2 แผนการทดลองใช้: กลุ่มเป้าหมายคือคณะครู ผู้บริหาร และนักเรียนในโรงเรียนหารเทารังสีประชาสรรค์ สพม.พัทลุง ระยะเวลาในการทดสอบ (UAT) และการใช้งานจริงจะอยู่ในช่วงเดือนที่ 5-6 ของโครงการ วิธีการเก็บข้อมูลทำได้โดยการใช้ API ดึงสถิติการมาเรียนและเกรดเฉลี่ยจากระบบส่วนกลาง
  • 4.2.3 แผนการประเมินผลและตัวชี้วัดความสำเร็จ: เครื่องมือวัดผลอ้างอิงจากข้อมูล KPI สำคัญที่ผู้บริหารสามารถเรียกดูได้แบบ Real-time ตัวชี้วัดเชิงปริมาณได้แก่ การลดเวลาเข้าถึงข้อมูลเหลือ 30 วินาที และลดความผิดพลาดการตัดสินใจของผู้บริหารร้อยละ 90 วิเคราะห์ข้อมูลโดยใช้โมเดล AI ในการทำนายและสรุปผล สำหรับแนวทางบริหารจัดการในระยะยาว มีแผนขยายเป็น Open API สำหรับกลุ่มโรงเรียนเครือข่ายในปีการศึกษาถัดไปเพื่อสร้างมาตรฐานร่วมกัน

4.3 ความสอดคล้องของนวัตกรรมกับวัตถุประสงค์ตามบริบทของสถานศึกษา (10 คะแนน)

  • 4.3.1 ความเชื่อมโยงกับวัตถุประสงค์ของโครงการ:
    1. ช่วยลดภาระงานเอกสารเพื่อให้ครูที่ปรึกษามีเวลาทำวิจัยในชั้นเรียน (CAR) และดูแลนักเรียนได้ใกล้ชิดขึ้น
    2. กระตุ้นให้เกิดวัฒนธรรมการใช้ Data-Driven ในการทำงาน
    3. สร้างรากฐานสู่การเป็น Smart School อย่างแท้จริง
  • 4.3.2 ความเหมาะสมกับบริบทและโครงสร้างการบริหารของสถานศึกษา:
    1) เหมาะสมกับทรัพยากรที่มีอยู่เนื่องจากได้รับการสนับสนุน Server และ Google Workspace แบบ Enterprise จากทางโรงเรียนเรียบร้อยแล้ว
    2) สามารถใช้งานจริงโดยไม่เพิ่มภาระ เพราะระบบจะทำงานประมวลผลอัตโนมัติ (Automated Insights) โดยไม่ต้องคีย์ข้อมูลซ้ำซ้อน
    3) โครงการนี้ยังมีศักยภาพในการขยายผลให้สอดคล้องกับขนาดของเครือข่าย โดยมีเป้าหมายทำระบบเป็น Open API สำหรับกลุ่มโรงเรียน สพม.พัทลุง เพื่อสร้างมาตรฐานข้อมูลร่วมกันในปีหน้า

ส่วนที่ 5 ความพร้อมและศักยภาพของสถานศึกษา (18 คะแนน)

5.1 ความพร้อมของผู้บริหาร ครูและบุคลากรทางการศึกษา (9 คะแนน)

  • 5.1.1 บทบาทของผู้บริหารในการสนับสนุนโครงการ: ผู้บริหารมีนโยบายส่งเสริมการใช้เทคโนโลยีเพื่อลดภาระงานเอกสารของครู และสนับสนุนการนำข้อมูลสารสนเทศมาใช้บริหารจัดการสถานศึกษา โรงเรียนให้การสนับสนุนทรัพยากรพื้นฐานที่มีอยู่แล้ว ได้แก่ พื้นที่ Server ของโรงเรียน และระบบ Google Workspace แบบ Enterprise เพื่อใช้ในการพัฒนาระบบ ผู้บริหารพร้อมให้ความร่วมมือในการทดลองใช้งานระบบ Dashboard เพื่อดูข้อมูลภาพรวม (KPI) ประกอบการบริหารจัดการสถานศึกษา
  • 5.1.2 ศักยภาพของครูผู้ดำเนินโครงการ: ทีมพัฒนามีพื้นฐานทักษะด้านเทคโนโลยี โดยได้ผ่านการอบรมหลักสูตร Python for AI และสามารถใช้งาน Google Apps Script ในการดึงและจัดการข้อมูลเบื้องต้นได้ ครูมีประสบการณ์ในการใช้งานแพลตฟอร์มการศึกษาพื้นฐานของกระทรวงฯ เช่น ระบบ SGS และระบบ DMC ทำให้เข้าใจโครงสร้างข้อมูลนักเรียนเป็นอย่างดี มีความมุ่งมั่นที่จะนำเทคโนโลยีมาช่วยลดเวลาการทำเอกสารซ้ำซ้อนเพื่อให้มีเวลาในการทำวิจัยในชั้นเรียน (CAR) และดูแลนักเรียนได้ใกล้ชิดขึ้น
  • 5.1.3 การมีส่วนร่วมของครูและบุคลากรในสถานศึกษา: มีการจัดตั้งทีมงาน (Team: Bad GateWay) ที่ประกอบด้วยครูที่ปรึกษาและนักเรียนแกนนำร่วมกันพัฒนาโครงการ ได้รับข้อมูลสะท้อนปัญหา (Pain Point) จากกลุ่มครูที่ปรึกษาที่ต้องใช้เวลาทำเอกสารกว่า 8 ชั่วโมง/สัปดาห์ ทำให้สามารถออกแบบระบบได้ตรงกับความต้องการของครูส่วนใหญ่ในโรงเรียน

5.2 ความพร้อมด้านโครงสร้างพื้นฐาน เทคโนโลยี และทรัพยากรสนับสนุน (9 คะแนน)

  • 5.2.1 โครงสร้างพื้นฐานด้านเทคโนโลยี: โรงเรียนมีระบบเครือข่ายอินเทอร์เน็ตพื้นฐานที่ครอบคลุมพื้นที่ห้องปฏิบัติการและห้องพักครู ซึ่งเพียงพอต่อการดึงข้อมูลและอัปเดต Dashboard อาศัยอุปกรณ์ดิจิทัลพื้นฐานที่มีอยู่ เช่น คอมพิวเตอร์ส่วนกลางของโรงเรียน และคอมพิวเตอร์พกพาประจำตัวของครูในการใช้งานระบบ ใช้พื้นที่ห้องปฏิบัติการคอมพิวเตอร์ของโรงเรียนเป็นศูนย์กลางในการพัฒนาระบบและทดสอบการทำงาน
  • 5.2.2 ระบบและเครื่องมือที่ใช้สนับสนุนการพัฒนานวัตกรรมปัญญาประดิษฐ์ (AI): ใช้แพลตฟอร์มและเครื่องมือที่ไม่เสียค่าใช้จ่ายเพิ่มเติม เช่น ภาษา Python สำหรับสร้างโมเดล AI เบื้องต้น, Google Apps Script สำหรับดึงข้อมูลอัตโนมัติ และ Looker Studio สำหรับทำ Dashboard ใช้ระบบฐานข้อมูลหลักที่โรงเรียนใช้งานอยู่แล้วในปัจจุบัน คือ ข้อมูลจากระบบ SGS และ DMC
  • 5.2.3 งบประมาณและทรัพยากรสนับสนุน: โครงการนี้ไม่ต้องพึ่งพางบประมาณจัดซื้อเพิ่มเติม เนื่องจากเป็นการใช้ทรัพยากรเดิมที่โรงเรียนสนับสนุนอยู่แล้ว คือ Server และ Google Workspace หากการพัฒนาระบบในโรงเรียนสำเร็จ มีแผนที่จะจัดทำเป็นรูปแบบ Open API เพื่อให้กลุ่มโรงเรียนเครือข่าย สพม.พัทลุง นำไปใช้งานได้ฟรีในปีการศึกษาหน้า เพื่อให้เกิดความคุ้มค่าและยั่งยืน
  • 5.2.4 ความมั่นคงปลอดภัยและการคุ้มครองข้อมูล: ข้อมูลนักเรียนถูกจัดการและจัดเก็บภายใต้ระบบ Server ของโรงเรียนและ Google Workspace ที่มีมาตรฐานความปลอดภัยของระบบสถานศึกษา มีการจำกัดสิทธิ์การเข้าถึงข้อมูลตามหน้าที่ โดยผู้บริหารสามารถดูภาพรวมสรุปผลได้ และครูที่ปรึกษาจะเข้าถึงได้เฉพาะข้อมูลของนักเรียนในความดูแลของตนเองเท่านั้น การดึงข้อมูลผ่าน Data Pipeline จากระบบ SGS / DMC จะดำเนินการแบบปิดเฉพาะผู้มีสิทธิ์ในโดเมนของโรงเรียน เพื่อป้องกันข้อมูลรั่วไหลออกสู่สาธารณะ
หน้า 6

ส่วนที่ 6 ผลลัพธ์ที่คาดว่าจะเกิดขึ้น (18 คะแนน)

6.1 ด้านการกำหนดนโยบายและทิศทางเชิงกลยุทธ์ของสถานศึกษา (6 คะแนน)

  • 6.1.1 ความสอดคล้องกับวิสัยทัศน์และยุทธศาสตร์ของสถานศึกษา: โครงการนี้สอดคล้องกับเป้าหมายการพัฒนาคุณภาพการศึกษาในการสร้างรากฐานสู่การเป็น Smart School อย่างแท้จริง โดยส่งเสริมวัฒนธรรมการใช้ Data-Driven ในการทำงาน เป้าหมายเชิงปริมาณที่คาดหวังคือการลดเวลาการเข้าถึงข้อมูลจาก 1 ชั่วโมง เหลือเพียง 30 วินาที และช่วยลดความผิดพลาดในการตัดสินใจเชิงนโยบายของผู้บริหารได้ร้อยละ 90
  • 6.1.2 การยกระดับคุณภาพผู้เรียนผ่านระบบบริหารจัดการ: คุณภาพผู้เรียนจะได้รับการดูแลอย่างทันท่วงที ลดอัตราการหลุดออกจากระบบการศึกษา เนื่องจากระบบ AI (Predictive Modeling) จะทำนายกลุ่มเสี่ยงล่วงหน้า ข้อมูลที่ใช้ติดตามประกอบด้วยสถิติการมาเรียนและผลสัมฤทธิ์ (เกรดเฉลี่ย) ซึ่งดึงมาจากระบบ SGS และ DMC ของสถานศึกษา
  • 6.1.3 การตัดสินใจบนฐานข้อมูล: ผู้บริหารสามารถใช้ข้อมูล KPI สำคัญบน Dashboard แบบ Real-time เพื่อวิเคราะห์และแก้ไขปัญหาได้อย่างรวดเร็ว แทนระบบเดิมที่ผู้บริหารจะมองเห็นภาพรวมปัญหาผู้เรียนล่าช้าเกิน 1 เดือน มีกลไกติดตามผ่าน Automated Insights ที่สรุปผลสารสนเทศระดับโรงเรียนให้อัตโนมัติ โดยอัปเดตข้อมูลตามรอบเวลาจริง

6.2 ด้านการบริหารทรัพยากรและการพัฒนาศักยภาพบุคลากร (6 คะแนน)

  • 6.2.1 การบริหารทรัพยากรอย่างคุ้มค่าและเหมาะสมกับบริบทของสถานศึกษา: โครงการเน้นการจัดสรรและใช้ประโยชน์จากทรัพยากรที่มีอยู่แล้วอย่างเต็มประสิทธิภาพ ได้แก่ Server ของโรงเรียน และระบบ Google Workspace แบบ Enterprise มีความคุ้มค่าและเป็นไปได้สูงมาก เนื่องจากพัฒนาด้วยเครื่องมือที่ไม่ต้องเสียค่าใช้จ่ายเพิ่มเติม (เช่น Python, Google Apps Script) และลดภาระงานซ้ำซ้อนได้จริง
  • 6.2.2 การพัฒนาศักยภาพบุคลากรและผู้นำการเปลี่ยนแปลง: เกิดกระบวนการแลกเปลี่ยนเรียนรู้ (PLC) ระหว่างครูที่ปรึกษาผู้ใช้งาน และทีมพัฒนาระบบ เพื่อปรับปรุงเครื่องมือให้ตอบโจทย์การทำงานจริงในโรงเรียน เกิดทีมแกนนำนวัตกรรม (Team: Bad GateWay) ซึ่งประกอบด้วยนักเรียนและครูที่ปรึกษา เป็นกลไกสำคัญในการขับเคลื่อนโครงการ
  • 6.2.3 การประยุกต์ใช้นวัตกรรมปัญญาประดิษฐ์ (AI) ในระบบบริหารจัดการ: มีการใช้ AI ในการวิเคราะห์ข้อมูลนักเรียนเพื่อค้นหากลุ่มเสี่ยง (Predictive Modeling) และใช้ระบบสร้างรายงานอัตโนมัติ (Automated Insights) ผลลัพธ์ที่ได้คือครูที่ปรึกษาสามารถลดภาระงานเอกสาร (จากเดิมกว่า 8 ชม./สัปดาห์) ทำให้มีเวลาคืนสู่ห้องเรียน ทำวิจัยในชั้นเรียน (CAR) และดูแลนักเรียนได้ใกล้ชิดขึ้น
หน้า 7

6.3 ด้านการพัฒนาระบบบริหารจัดการและการเปลี่ยนแปลงองค์กร (6 คะแนน)

  • 6.3.1 การออกแบบระบบหรือกระบวนการทำงานใหม่: กระบวนการทำงานเปลี่ยนจากการจัดการข้อมูลผ่านรายงานกระดาษและ Checklist มาเป็นการใช้ Data Pipeline เชื่อมข้อมูลอัตโนมัติ และแสดงผลบน Looker Studio Dashboard ความแตกต่างจากแนวปฏิบัติเดิมคือ การเปลี่ยนผ่านจากการวิเคราะห์ข้อมูล "แบบย้อนหลังและซ้ำซ้อน" สู่การบริหารจัดการแบบ "Real-time"
  • 6.3.2 การขยายผลและการบูรณาการทั้งสถานศึกษา: เมื่อทดลองใช้และปรับปรุงระบบ (UAT) เสร็จสิ้น มีแนวทางพัฒนาต่อยอดเป็น Open API เพื่อขยายผลให้กับกลุ่มโรงเรียนเครือข่าย สพม.พัทลุง ในปีการศึกษาหน้า ระบบนี้จะถูกบูรณาการเข้าเป็นส่วนหนึ่งของการติดตามดูแลนักเรียนประจำวันและประจำภาคเรียน เพื่อสร้างมาตรฐานการจัดการข้อมูลร่วมกันอย่างยั่งยืน

ส่วนที่ 7 แนวทางการขยายผลและความยั่งยืน (9 คะแนน)

7.1 แผนการขยายผลภายในสถานศึกษา (3 คะแนน)

  • 7.1.1 การขยายผลเชิงระบบงานบริหารจัดการ:
    - ระบบของกลุ่มงานที่มีแผนขยายผล: ในระยะเริ่มต้นโครงการมุ่งเน้นที่ระบบดูแลช่วยเหลือนักเรียน (สำหรับครูที่ปรึกษาและผู้บริหาร) เมื่อระบบเสถียรแล้ว มีแผนขยายผลไปยัง งานวัดผลและวิชาการ (เพื่อวิเคราะห์แนวโน้มผลสัมฤทธิ์รายวิชา) และ งานกิจการนักเรียน (เพื่อประมวลผลสถิติพฤติกรรมและการเข้าร่วมกิจกรรม)
    - แนวทางการปรับกระบวนการทำงาน: ยกเลิกการใช้แบบบันทึกหรือ Checklist แบบกระดาษในขั้นตอนการรายงานผลประจำเดือน โดยปรับให้ใช้ข้อมูลสารสนเทศที่ประมวลผลผ่านระบบ EduSmart Dashboard เป็นหลักฐานอ้างอิงในการประชุมผู้บริหารและครู
  • 7.1.2 การพัฒนาศักยภาพบุคลากรและทีมขับเคลื่อน:
    - แผนการพัฒนาบุคลากร: จัดอบรมเชิงปฏิบัติการ (Workshop) ภายในโรงเรียนแบบเพื่อนช่วยเพื่อน (PLC) แนะนำการเข้าถึงและการอ่านข้อมูลจาก Dashboard ให้แก่ครูที่ปรึกษาเพื่อให้สามารถนำข้อมูลไปใช้ดูแลนักเรียนได้อย่างมีประสิทธิภาพ
    - การจัดตั้งคณะทำงาน: จัดตั้ง "ทีมสารสนเทศอัจฉริยะ (Smart Data Team)" โดยมีตัวแทนจากฝ่ายบริหาร ครูคอมพิวเตอร์ และมีกลุ่มนักเรียนแกนนำ (Team: Bad GateWay) เป็นผู้ช่วยสนับสนุนด้านเทคนิคในการดูแลรักษาระบบร่วมด้วย

7.2 แผนการขยายผลสู่เครือข่ายภายในหรือภายนอกสถานศึกษา (3 คะแนน)

  • 7.2.1 การเผยแพร่และถอดบทเรียนเชิงบริหาร: ถอดบทเรียนกระบวนการทำงานจัดทำเป็นคู่มือ "Smart School Data Framework ฉบับปฏิบัติจริง” เพื่อสรุปขั้นตอนการสร้าง Dashboard พื้นฐานสำหรับโรงเรียน เผยแพร่ผ่านเว็บไซต์ของโรงเรียนหารเทารังสีประชาสรรค์ (https://www.harnthao.ac.th)
  • 7.2.2 การสร้างความร่วมมือเชิงระบบ: เครือข่ายที่คาดว่าจะร่วมขยายผลคือกลุ่มโรงเรียนเครือข่ายในสังกัด สพม.พัทลุง โครงการมีแผนพัฒนาโมเดลนี้ให้เป็นรูปแบบ Open API เพื่อให้โรงเรียนเครือข่ายสามารถนำไปเชื่อมต่อและใช้งานได้ สร้างมาตรฐานการจัดการข้อมูลร่วมกันในปีการศึกษาหน้า

7.3 แนวทางสร้างความยั่งยืนของนวัตกรรม (3 คะแนน)

  • 7.3.1 การบูรณาการเข้าสู่แผนและนโยบายสถานศึกษา: นำระบบนวัตกรรมปัญญาประดิษฐ์นี้บรรจุเป็นส่วนหนึ่งของ "โครงการพัฒนาระบบสารสนเทศและดูแลช่วยเหลือนักเรียน" ในแผนปฏิบัติการประจำปีของสถานศึกษา เพื่อให้มีโครงสร้างรองรับการทำงานอย่างเป็นทางการ
  • 7.3.2 การจัดสรรทรัพยากรและงบประมาณอย่างต่อเนื่อง: เน้นการใช้ทรัพยากรพื้นฐานที่โรงเรียนมีอยู่แล้วให้เกิดความคุ้มค่าสูงสุด ได้แก่ ระบบเครือข่าย Server และ Google Workspace แบบ Enterprise ของโรงเรียน ทำให้ระบบสามารถดำเนินการต่อไปได้ในระยะยาวโดยไม่ต้องพึ่งพางบประมาณจัดซื้อซอฟต์แวร์เพิ่มเติมจำนวนมาก
  • 7.3.3 ระบบติดตาม ประเมินผล และปรับปรุงพัฒนา: กรอบระยะเวลาดำเนินการประเมินผลความพึงพอใจและประสิทธิภาพการใช้งานระบบทุกสิ้นภาคเรียน กลไกการสะท้อนผลใช้กระบวนการ PLC ของกลุ่มครูที่ปรึกษาเป็นเวทีในการรับฟังข้อเสนอแนะ (Feedback) เพื่อนำมาปรับปรุงหน้าตาและฟีเจอร์ของ Dashboard ให้ตอบโจทย์การใช้งานจริงอยู่เสมอ
  • 7.3.4 การกำกับดูแลด้านจริยธรรม ความปลอดภัย และการใช้ข้อมูล: ปฏิบัติตาม พ.ร.บ. คุ้มครองข้อมูลส่วนบุคคล (PDPA) อย่างเคร่งครัด ผลการประมวลผลกลุ่มเสี่ยงจากโมเดล AI (Predictive Modeling) จะถูกเก็บเป็นความลับระดับบุคคล โดยกำหนดสิทธิ์เข้าถึงข้อมูลให้เฉพาะครูที่ปรึกษาประจำชั้นและผู้บริหารระดับสูงเท่านั้น เพื่อป้องกันการตีตรา (Labeling) หรือการนำข้อมูลนักเรียนไปใช้ในทางที่ผิด

ภาคผนวก (ถ้ามี)

  • แผนผังระบบ
  • โครงสร้างฐานข้อมูล
  • ตัวอย่างระบบที่พัฒนา เช่น Dashboard ฯลฯ
  • แผนการประเมินผล
  • เอกสารสนับสนุนอื่น ๆ
